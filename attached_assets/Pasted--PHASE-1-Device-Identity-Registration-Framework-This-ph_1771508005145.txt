🧱 ARCHITECTURE — PHASE 1
Local Identity Structure

Stored at:

~/.joincloud/system/identity.json


Example:

{
  "host_uuid": "b7c19f8a...",
  "installation_id": "uuid-v4",
  "created_at": 1735689600,
  "last_seen_at": 1735690100,
  "version": "1.0.0",
  "registration_status": "pending"
}

host_uuid Generation (Safe)
host_uuid = sha256(
  machineId +
  installationSalt
)


Where:

machineId = OS-level machine ID (hashed locally)

installationSalt = random UUIDv4 (generated once)

Never transmit raw machineId

Registration API (Outbound Only)
POST /api/v1/hosts/register


Payload:

{
  "host_uuid": "...",
  "installation_id": "...",
  "version": "1.0.0",
  "platform": "macos",
  "arch": "arm64",
  "first_installed_at": 1735689600
}


Response:

{
  "status": "registered",
  "server_time": 1735689650
}


If offline:

Queue payload

Retry every 6 hours

Never block UI

Heartbeat Endpoint
POST /api/v1/hosts/heartbeat


Payload:

{
  "host_uuid": "...",
  "version": "1.0.0",
  "uptime_seconds": 4220
}


Admin panel updates last_seen_at.

Heartbeat interval:

Every 12 hours

Random jitter ± 10%

Never during active transfer

🧠 PHASE 1 — JOINCLOUD (ELECTRON) IMPLEMENTATION PROMPT
Implement Phase 1: Device Identity & Registration Module in JoinCloud Electron App.

Objectives:
- Generate stable host_uuid.
- Persist identity.
- Attempt outbound registration.
- Queue if offline.
- Implement heartbeat mechanism.

Requirements:

1. Create module: /core/device-identity
   - Generate installationSalt (UUIDv4)
   - Fetch OS machineId (hashed locally)
   - Compute host_uuid = sha256(machineId + installationSalt)
   - Store in ~/.joincloud/system/identity.json

2. Create module: /core/registration-client
   - Method: registerHost()
   - Method: sendHeartbeat()
   - Use HTTPS POST
   - Timeout 2s
   - Non-blocking
   - Retry on failure (exponential backoff)

3. Offline Handling:
   - If registration fails, store pending payload locally.
   - Retry every 6 hours.
   - Do not block app startup.

4. Heartbeat:
   - Trigger every 12 hours (with ±10% jitter).
   - Skip during active file transfers.
   - Must not interfere with streaming pipeline.

5. Logging:
   - Log registration success/failure.
   - Do not log hardware identifiers.
   - No console spam.

6. Security:
   - Do not send raw machineId.
   - Do not send LAN IP.
   - Use certificate validation.
   - Handle server errors gracefully.

7. Performance:
   - Must not delay app launch > 100ms.
   - No synchronous filesystem blocking in hot path.

Do NOT push until verified locally.

Verification Checklist:
- Fresh install generates identity.json.
- Re-launch does not regenerate host_uuid.
- Registration succeeds when online.
- Offline install queues properly.
- Heartbeat updates correctly.
- No impact on LAN file transfer performance.
- No crash if server unreachable.
