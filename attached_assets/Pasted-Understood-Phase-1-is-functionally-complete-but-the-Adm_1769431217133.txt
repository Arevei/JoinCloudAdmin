## 📊 1️⃣ Dashboard (No Logic Change)

### Purpose

System-level overview (what exists today).

### What to Show

Keep **exactly what the current main screen shows**, including:

* Total installs
* Active users (7d)
* Avg uptime
* Total data processed
* Shares (LAN vs Public)
* Charts / trends

### Changes Required

* Move existing dashboard UI under **Dashboard** menu
* No new metrics required

---

## 👥 2️⃣ Users (Device-Centric View)

### Purpose

Operate JoinCloud **per user/device**, not as raw analytics.

---

### Users List Page

Each **User Card / Row** represents **one deviceUUID**.

Show on each card:

* deviceUUID (shortened)
* deviceIndex
* Platform (macOS)
* App version
* Last seen
* Heartbeat status (● green / yellow / red)

Each card must have **two clear action buttons**:

1. **User Analytics**
2. **Support**

---

### User Analytics View (Per User)

When clicking **User Analytics**:

Show analytics **only for that user**, such as:

* Uptime (daily / total)
* Files uploaded
* Files shared
* Public vs LAN shares
* Activity timeline

This is **filtered reuse of existing analytics**, not new computation.

---

### Support Shortcut (Per User)

Clicking **Support** from a user card:

* Opens the **Support thread** for that user
* Same as Support section, but pre-filtered

---

## 🏆 3️⃣ Leaderboard (Engagement Ranking)

### Purpose

Identify **most active users** quickly.

---

### Leaderboard View

Show ranked lists (tabs or sections):

* Top users by **Uptime**
* Top users by **Files Uploaded**
* Top users by **Shares Created**

Each leaderboard entry shows:

* Rank
* deviceIndex / short UUID
* Metric value
* Quick “View User” action

Leaderboard uses **existing aggregated data** only.

No gamification, no scoring system.

---

## 💬 4️⃣ Support (Conversation-Centric)

### Purpose

Act as a **support inbox**, not a user list.

---

### Support Main View

Show **user cards only when messages exist**.

Each card shows:

* deviceUUID / deviceIndex
* Last message snippet
* Last activity time
* Unread indicator (if admin hasnels)

Clicking a card opens:

* Full message history
* Admin reply box (text-only)

---

### Behavior Rules

* Messages are grouped **by user**
* Order by most recent activity
* No deletion yet
* No status automation yet
